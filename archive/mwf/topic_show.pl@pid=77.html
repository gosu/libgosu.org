By oli.obk Date 2008-11-26 11:49 Edited 2008-11-26 12:23
isn't cairo for fonts? Sorry, don't know cairo

Gosu supports easy integration of rmagick, so if you have a lot of unchanging polygons (meaning they won't change for a long enough time to make the image generation feasible), that'll be what you want.

Colored lines, triangles and sqares are currently implemented in the Graphics class. But there's no ruby way to texture them afaik.

Julian mentioned on irc that user defined opengl operation should get Gosu support some time in the future. Once those exist, it'll be easy creating any kind of drawing operation you want, even dynamic polygons. Before that, it's hacky ;)

anti-aliasing is a linux-opengl problem (I just got a 20 line program that draws antialiased stuff on windows... if you have the hardware that supports it).

edit: ok, I looked up cairo, cairo will only create images for you, you'd still have to move them to a texture (which is easy in c++). Using ruby it's probably even easier, just as mentioned above for rmagick, doing so every frame will probably eat away at your fps.
