By Spooner Date 2012-02-15 14:45
If you are using Chingu collision, then Chingu only stores lists of objects by class (so if you collide with ). It could add the functionality you require without a huge amount of extra code (and no slowdown), but the author doesn't maintain it any more.

I'd recommend you store objects yourself, so you don't have to rely on Chingu:

Rather than:
GiantFish.create
AlienFrog.create
@player.each_collision Enemy do...

Try:
@enemies << GiantFish.new # Subclass of Enemy
@enemies << AlienFrog.new # Subclass of Enemy

@player.each_collision @enemies do...

Actually, you can still use #create in the second version if you want Chingu to manage drawing and updating.
