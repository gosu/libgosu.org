By RavensKrag Date 2010-05-24 00:48
I'm working with the ruby version of Gosu, and I'm making a game where the different elements of a character are stored as separate sprite sheets in different images, and combined together at runtime.

Thus, I want to know if there any way to call Gosu::Image::load_tiles on a Gosu::Image.  The documentation says that it is possible to use Gosu::Image::load_tiles on a RMagick image, but I'm using Texplay to combine two images together (splice) for I have heard this is faster. 

Would I have to convert the Gosu::Image into a RMagick image somehow first? Or Is there some completely different way to go about it?
By jlnr (dev) Date 2010-05-24 12:19
I have no experience with TexPlay, but technically Gosu doesn't check if the "RMagick image" is really an RMagick image. If you can get TexPlay to respond to to_blob with a binary string of 32-bit ARGB tuples, you are set. ;)

By banister Date 2010-05-26 05:16 Edited 2010-05-26 05:21
you dont need to call #to_blob on it, you just need to pass in the Gosu::Image. But i just remembered that i didn't implement #rows and #columns methods so it probably still wont work unless you do the following:

class Gosu::Image
    alias_method :rows, :height
    alias_method :columns, :width
end

and change your last line to:
@sprites = Gosu::Image::load_tiles(self, @body, 40, 80, false)

Can you try this first and tell me if it works? If it does work, i'll add it in the next release of TP so this functionality should work out of the box
