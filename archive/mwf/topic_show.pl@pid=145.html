By Basic Date 2009-01-04 13:04
I think I've found the reason for it. I ran into again, while playing around with chipmunk, I was basically copying the falling blocks gosu/chipmunk example when it happened. so I had a reference to see what was different in mine compared to Leonardo Boiko's version, which wasn't crashing on my machine.

I was being lazy and referencing the colour from the gosu module directly into the draw methods

@window.draw_line(@vector[p][:x] + self.x, @vector[p][:y] + self.y, Gosu::red, @vector[p - 1][:x] + self.x, @vector[p - 1][:y] + self.y, Gosu::red, 1)

but when I move that into an instance var
@colour = Gosu::red # in def initialize

@window.draw_line(@vector[p][:x] + self.x, @vector[p][:y] + self.y, @colour, @vector[p - 1][:x] + self.x, @vector[p - 1][:y] + self.y, @colour, 1)

the problem disappears, and the app does not :)
