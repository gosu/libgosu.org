By jlnr (dev) Date 2009-05-03 19:37 Edited 2009-05-03 19:47
There's two bugs here.

First, the crash, which I have never ever seen so far. It seems that Ruby 1.8.7 is to blame, or rather, that SWIG needs to catch up so Gosu will work with that particular version.

It seems to happen with all native libraries that have been packaged using SWIG with object tracking enabled (http://www.ruby-forum.com/topic/161876):

> No, it's not your fault, and it's not our fault either. With 1.8.7, the
> ruby developers chose to change both the public APIs and aspects of the
> GC/memory management internals. This broke any SWIG-generated ruby
> libraries that use certain SWIG features (object tracking). IMO this was
> a sloppy decision on a stable branch that reflects badly on ruby's
> development process, but - there we are.


What you can do here is to wait for the next version, which I will generate using the newer SWIG 1.3.3.9. Until then, you will probably have to use either Ruby 1.8.6 or 1.9, both of which seem not to have this feature.

---

The second one is a bit weirder. What is happening is that Gosu creates all its textures, then shows the game; as soon as you collect a star, the internal texture is updated on-demand to include the rendered digit "1". For some reason, the latest Ubuntu drops *everything* that was drawn to the texture earlier in that moment. I have no idea why, but if this is a widespread problem there are obvious (but horribly ugly) workarounds for that :|

A bit unfortunate that I can't reproduce it in my VM, but i'll try that on a friend's Ubuntu laptop.
