By jlnr (dev) Date 2008-12-20 02:33
Ah, I didn't realize that you participated in the last LD :) One more game I can play! (No time to borrow a Windows machine right now ...)

Regarding the first issue, I don't think Gosu is doing anything unusual in Sample::play so I wonder if it's a problem with SDL_mixer or its backing audio system. You can make the gem version of Gosu use FMOD (v3) by having it installed globally, which means at /usr/lib/libfmod.so, because the Gosu configure script checks for exactly this location before falling back to SDL_mixer. The fmod.h header and the errors one also have to be available to gcc without any special flags.
This is pretty ugly and not a proper solution--but it this helps it would mean that SDL_mixer is the problem. Not sure what else to try for debugging this :)

As for the second problem, I looked at Jan's oldish Pango/Freetype port of Gosu's text system and it doesn't look like it's caching as much as it could. Maybe there's something in there that used to be more lightweight but shouldn't be initialized multiple times per frame nowadays. Since the next version of Gosu was supposed to be about graphics and performance I'll give it a try to cache as much as possible there. I'll bump this thread when I get to it.

In any case, thanks for the feedback! I only test the Linux version in a VM so I can hardly judge the performance of it :)
