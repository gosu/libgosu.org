require 'gosu';

class Box
  attr_reader :x;
  attr_reader :y;

  def initialize
    @x = 0;
    @y = 0;
    @vel_x = 0.5;
    @vel_y = 0;
    @graphic = Gosu::Image.new("media/box.png");
  end

  def move
    @x += @vel_x
  end;

  def draw
    @graphic.draw(@x, @y, 1);
  end;
end;

class Window < Gosu::Window
  def initialize
    super(640, 480, false);

    @box = Box.new;
    @beginMovement = false;
  end;

  def button_down(id)

  end;

  def update

  end
end;

By nietzschette Date 2016-02-22 06:46
That is exactly the sort of thing the update method is intended to handle!  In fact, your code already looks pretty close to optimum, though there are a few things I would do differently because I tend to have my boxes move in more than one direction, often with a layer of intelligence.  If these boxes are only suppose to move right, than you might as well not bother with a method call to box.move because it's only one line anyway.  Reducing method call is probably your greatest speed up.  Those booleans could probably be trimmed as well, though I'd wait until testing with 100-1000 boxes before making assumptions about how the code executes.

I like to use enumerators and arrays where I can.  your entire box object could simply be packed into an array like this:

[Gosu::Image,x,y,vx,vy]

thus eliminating the overhead associated with the object.  Better yet, if you want many boxes to share the same graphic, hold just the coordinates in a separate, nested array

@graphic = Gosu::Image
@boxes = [
[x0,y0,vx0,vy0],
[x1,y1,vx1,vy1],
...
]

and enumerate through @boxes for movement and draw operations, so you don't need to allocate memory for separate box classes or spend precious time looking-up method calls for those objects.

def update
@boxes.each{|b|
   if b[0] >= 100
    b[0] += 0.5
   end
}
end

def draw
@boxes.each{|b|
  @graphic.draw[b[0],b[1],1]
}
end

By jlnr (dev) Date 2016-02-22 16:20
Yep, exactly. A few minor nitpicks:
- You don't any of these semicolons in Ruby :)
- In draw, you want to iterate over the boxes and .draw each of them. You can use a for ... in loop for that, but @box.each { |b| b.draw } is more common in Ruby. It can even be shortened to @box.each(&:draw).
- Method names usually use_underscores() instead of javaCase()
- You don't need to pass false to super in initialize anymore, that's Gosu's old interface (which still works). The new way is super(w, h) or super(w, h, fullscreen: some_boolean)
- In button_down, you can use button_id_to_char to see which number was pressed:
  def button_down(id)
    ch = Gosu::button_id_to_char(id)
    if ch =~ /^[0-9]$/ then # Regexp to test for 0-9
      index = (ch.to_i + 1) % 10
      box[index].move = true
    end
  end
That has the advantage of also working with the numpad.
