Works for me on Ubuntu. A quick Google suggests:

> "The ABI changed between 2.95 and 3.x. This indicates you are using
> the older library with code produced by a newer compiler.".  However, I don't

(http://groups.google.com/group/comp.sys.hp.mpe/browse_thread/thread/27d5e20cb9aad94e/96b056a13e00a6ab)

I would think that either (a) Ruby or a part of it has been compiled with a different compiler/libstdc++ than Gosu, or (b) no C++ library is found at all and this just happens to be the first error to be found.

I have no idea how Fedora works, but you can give a clean installation of Ruby via RVM a try.
